# GET /api/communications/v2/news

**Получение новостей портала продавцов{{ /api/communications/v2/news }}**

теги: `API новостей`

**Полный путь:** `GET /api/communications/v2/news`

## Описание

<div class='description-title'><span>Описание метода</span></div>

Метод позволяет получать новости портала продавцов. <br> Для получения успешного ответа необходимо указать
один из параметров `from` или `fromID`. <br> За один запрос можно получить не более 100 новостей.

<div class="description_limit">
<a href='./api-information#tag/Vvedenie/Limity-zaprosov'>Лимит запросов</a> на один аккаунт продавца:


| Тип | Период | Лимит | Интервал | Всплеск |
| --- | --- | --- | --- | --- |
| Персональный | 1 мин | 1 запрос | 1 мин | 10 запросов |
| Сервисный | 1 мин | 1 запрос | 1 мин | 10 запросов |
| Базовый | 1 ч | 1 запрос | 1 ч | 1 запрос |
</div>

## Авторизация

- `HeaderApiKey` (scopes: —)

## Параметры

| Имя | В | Тип | Обязательный | Описание |
|---|---|---|---|---|
| `from` | query | string (date); пример: `2025-02-06` | нет | Дата, от которой необходимо выдать новости |
| `fromID` | query | integer (uint64) | нет | ID новости, начиная с которой — включая её — нужно получить список новостей |

## Ответы

    - `header` — string. Заголовок новости
    - `id` — integer. ID новости
    - `types` — array<object>. Теги новости
      - *(элементы)*
        - `id` — integer. ID тега
        - `name` — string. Название тега

- `status` — number. HTTP статус-код

- `timestamp` — string (date-time). Дата и время запроса
